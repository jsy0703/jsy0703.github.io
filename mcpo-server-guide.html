🐍 Python 직접 설치 방법

1단계: 저장소 클론

git clone https://github.com/open-webui/open-webui.git
cd open-webui

2단계: Python 가상환경 설정

python -m venv venv
source venv/bin/activate # Linux/Mac
venv\Scripts\activate # Windows

3단계: 종속성 설치

pip install -r requirements.txt
pip install -e .

4단계: 환경 변수 설정

export WEBUI_SECRET_KEY="your-secret-key"
export ENABLE_MCP=true

5단계: 서버 시작

python -m open_webui.main

🐳 Docker 설치 방법 (권장)

1단계: Docker 이미지 다운로드

docker pull ghcr.io/open-webui/open-webui:main

2단계: 컨테이너 실행

docker run -d --name open-webui \
-p 3000:8080 \
-e ENABLE_MCP=true \
-e WEBUI_SECRET_KEY="your-secret-key" \
-v open-webui:/app/backend/data \
ghcr.io/open-webui/open-webui:main
✅ 설치 완료! http://localhost:3000 에서 접속 가능합니다.

🔐 MCP (Model Context Protocol) 인증 설정

1단계: 관리자 계정 생성

첫 접속 시 관리자 계정을 생성합니다.

2단계: MCP 서버 설정

설정 → MCP 설정에서 다음 정보 입력:

서버 이름: atlassian-mcp
엔드포인트: https://your-mcp-server.com/mcp
API 키: your-api-key

3단계: Atlassian 연동 설정

{ "mcpServers": { "atlassian": { "command": "npx", "args": [ "-y", "@atlassianlabs/mcp-server-atlassian@latest" ], "env": { "ATLASSIAN_API_TOKEN": "your-atlassian-api-token", "ATLASSIAN_DOMAIN": "your-domain.atlassian.net", "ATLASSIAN_EMAIL": "your-email@example.com" } } } }

4단계: Microsoft Teams 연동

{ "mcpServers": { "teams": { "command": "node", "args": [ "path/to/teams-mcp-server.js" ], "env": { "TEAMS_CLIENT_ID": "your-teams-client-id", "TEAMS_CLIENT_SECRET": "your-teams-client-secret", "TEAMS_TENANT_ID": "your-tenant-id" } } } }

🔗 Open WebUI와 MCP 연결

1단계: MCP 클라이언트 설정

Open WebUI 설정에서 MCP 클라이언트를 활성화합니다.

2단계: 서버 연결 테스트

curl -X POST http://localhost:3000/api/mcp/test \
-H "Content-Type: application/json" \
-d '{"server": "atlassian"}'

3단계: 권한 설정

MCP 서버별로 접근 권한을 설정합니다:

  • 읽기 권한: 데이터 조회
  • 쓰기 권한: 데이터 수정
  • 관리자 권한: 서버 설정 변경

⚙️ 추가 설정

환경 변수 설정

# .env 파일 생성
WEBUI_SECRET_KEY=your-secret-key
ENABLE_MCP=true
MCP_SERVER_URL=http://localhost:8000
ATLASSIAN_API_TOKEN=your-token
TEAMS_CLIENT_ID=your-client-id

로그 설정

export LOG_LEVEL=INFO
export MCP_LOG_LEVEL=DEBUG

🛠️ 문제 해결

⚠️ 일반적인 문제

  • 포트 충돌: 다른 포트 사용 (-p 8080:8080)
  • 권한 오류: sudo 권한으로 실행
  • 메모리 부족: Docker 메모리 할당 증가
  • MCP 연결 실패: 네트워크 설정 확인

로그 확인

# Docker 로그 확인
docker logs open-webui

# MCP 서버 로그
tail -f /var/log/mcp-server.log

연결 테스트

# 웹 인터페이스 접속 테스트
curl http://localhost:3000/health

# MCP 서버 연결 테스트
telnet localhost 8000
